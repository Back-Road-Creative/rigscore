# Moat & Ship — Agent B release notes

## Scoring
- **Coverage-scaling regression fix (backlog 3.1).** Weight-0 advisory
  checks no longer affect the coverage-penalty denominator. Adding new
  advisory checks (like `documentation`) can no longer shift existing
  project scores. Characterization tests in
  `test/scoring-advisory-isolation.test.js` pin the new invariant.
- **CI `--fail-under` restored from 15 to 30** on the self-scan step.

## Profiles
- Two new built-in profiles: `home` (single-user dev boxes — governance
  / skill-files / MCP weighted, infra / docker / windows disabled) and
  `monorepo` (default weights + `--recursive --depth 3` hints).
- Existing profiles (`default`, `minimal`, `ci`) unchanged.
- `action.yml` input description, `--help` text, and README enumerate
  all 5 profiles. Full per-profile weight tables in `docs/profiles/`.

## CLI
- `--baseline <path>` — first run writes a `{timestamp, version,
  findings}` snapshot; subsequent runs report only new findings and
  exit 1 on any delta.
- `rigscore diff <baseline> <current>` — JSON diff subcommand; exit 1
  if new findings exist.
- `rigscore init [--profile <name>]` — write a commented
  `.rigscorerc.json` starter. Refuses to overwrite without `--force`.
- `rigscore explain <findingId>` — print the relevant
  `docs/checks/<checkId>.md`; targets the finding-specific section
  when available, with a helpful "available checks" error path.

## Suppression semantics
Three pattern forms for `suppress[]` entries and `--ignore`:
- **Substring** (legacy, backwards-compatible): `"env file found but NOT in .gitignore"`
- **Exact findingId**: `"claude-md/missing-claude-md"`
- **Glob** (any `*`): `"skill-files/drive-resume-*"`
- **Regex**: `"re:/.*sudo.*/i"` — matches findingId OR title, case-i default

## Config merging
`~/.rigscorerc.json` and project `.rigscorerc.json` now merge at load
time: arrays concatenate (suppressions, paths, disabled checks, etc.),
scalars use project-wins precedence. Users can keep personal
suppressions in `~/.rigscorerc.json` without losing them per-project.
