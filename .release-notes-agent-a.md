# Agent A — Moat & Ship release notes

Consolidated release notes for Agent C to fold into CHANGELOG.md.

## Check quality

- **`instruction-effectiveness`** — dead-file-reference false-positive reduction: strips `:line` / `:line-line` / `#L123-L456` suffixes before existence check; adds new `.rigscorerc.json` key `instructionEffectiveness.crossRepoRefs: [glob, ...]` to exempt sibling-repo refs; skips dead-ref scanning for `.claude/projects/<slug>/memory/`, `.claude/commands/*`, and `.claude/skills/*/evals/*` (these describe other projects by design); tightens `looksLikeFilePath` to reject bare extensions (`.md`, `.sh`), JS property-access strings (`data.filesDiscovered`), and versioned shell fragments (`python3`, `pip3`, `node22`). Dead-ref count on `/home/joe` drops from 143 (unsuppressed) to 12.
- **`skill-files`** — new `.rigscorerc.json` key `skillFiles.allowlist: [{ skill, pattern, reason }, ...]` — allowlists escalation/shell/exfil/persistence/indirect-injection/trust-exploitation patterns in operator skills by skill directory + pattern id (no title substring matching). Escalation patterns now emit stable pattern ids (`sudo`, `chmod-777`, `chmod-plus-x`, `disable-firewall`, etc.). All finding emissions in these families now carry `context: { file, patternId, skill }` so external suppression tooling can evaluate cleanly.

## Finding quality

- Every finding emits `findingId` (auto-generated `<checkId>/<slug>` if unset), and checks with fixers now set explicit `findingId` strings (`env-exposure/env-not-gitignored`, `env-exposure/env-world-readable`, `permissions-hygiene/ssh-dir-permissions`, `permissions-hygiene/ssh-key-permissions`, `permissions-hygiene/sensitive-file-world-readable`, `skill-files/world-writable`).
- New `evidence` field on findings — ≤120-char snippet of the offending content. Rendered by the terminal reporter as `Evidence: <snippet>`; surfaced in SARIF as `result.properties.evidence`.

## SARIF

- Per-finding ruleIds: every SARIF result now uses `<checkId>/<findingId-or-slug>` (previously check-level only). The bare `<checkId>` remains registered as a tool-component rule fallback so legacy consumers continue to resolve.
- Evidence field surfaced via `result.properties.evidence`.

## Fix matcher

- `src/fixer.js` now prefers `fixer.findingIds: string[]` equality over title-substring matching. Legacy `fixer.match(finding)` predicates still work as a fallback but emit a one-shot deprecation `console.warn` the first time they are consulted per process. All bundled fixers (`env-not-gitignored`, `env-world-readable`, `ssh-dir-permissions`, `ssh-key-permissions`, `gitignore-sensitive-patterns`, `skill-file-world-writable`) have been migrated to declare `findingIds`.
