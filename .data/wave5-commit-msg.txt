fix(scanner,checks,cli): close env-exposure ambiguity, slugify drift, dead ternary, shadow var, loadChecks race

Five independent quality/correctness flaws from the 2026-05-27 health
report. Each is small on its own but compounding — the slugify drift
and the env-exposure ambiguity both showed up as user-facing surprises.

1. (mcp-config) Inner `const registryResult` at line 538 shadowed the
   outer `registryResult` captured at line 221 (the MCP registry fetch
   result). The shadow made the downstream registry-fallback INFO
   finding evaluate the loop's last-iteration npm-finding object
   instead of the registry result. Renamed inner to `npmFinding`.

2. (src/index.js baseline branch) `process.exit(added.length > 0 ? 1 : 0)`
   was dead — the early-return at line 334 guarantees `added.length > 0`
   by the time the ternary runs. Replaced with `process.exit(1)` and
   added a comment citing the guard.

3. (env-exposure) When `.env`, `.env.local`, and `.env.production` were
   all present and only `.env` was gitignored, the finding said ".env
   file found but NOT in .gitignore" — generic and wrong; the real
   offenders were the others. Collect the unignored files by name,
   include them in title + evidence + remediation.

4. (utils + scanner + baseline) Scanner's local `slugify()` and
   baseline.js's inline slug expression were almost identical but
   baseline omitted the `replace(/^-|-$/g, '')` step, so titles with
   non-alphanumeric bookends produced divergent findingIds across the
   two paths. Extract canonical `slugify` to utils.js, import from
   both call sites, delete the inline copies.

5. (loadChecks) `_registeredFixes = {}` ran AFTER `await fs.promises.readdir`,
   so a downstream consumer could observe a stale populated cache
   during the I/O yield. Move the reset to entry, BEFORE any await.
   Change initial value to null so `getRegisteredFixes()` can throw a
   clear error when called before loadChecks (the previous silent
   empty-map return made `--fix` look like "no fixes available").

Closes Flaws #3, #5, #6, #7, #8.
